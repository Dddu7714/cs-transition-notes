# Python基础题目总结知识清单

## 一、输入输出基础
### 需要掌握
- `input()` 读取输入
- `print()` 输出结果
- `split()` 拆分一行数据
- `map()` 批量转换类型
- `int()`、`float()` 类型转换
- 多组测试数据的读法

### 常见写法
```python
a, b = map(int, input().split())
```

```python
n = int(input())
for _ in range(n):
    ...
```

### 易错点
- 输入读成字符串后没转整数
- 多组数据时少写循环
- 输出格式和题目要求不一致

---

## 二、条件判断与循环
### 需要掌握
- `if / elif / else`
- `for`
- `while`
- `break`
- `continue`

### 典型作用
- 分类讨论
- 重复处理数据
- 提前结束循环
- 跳过某些情况

### 易错点
- 条件顺序写错
- 死循环
- `break` 只跳出当前这一层循环

---

## 三、基础数值运算
### 需要掌握
- 加减乘除：`+ - * /`
- 整除：`//`
- 取余：`%`
- 幂运算：`**`
- 绝对值：`abs()`
- 最大最小值：`max()`、`min()`
- 四舍五入：`round()`
- 求和：`sum()`

### 典型题型
- 求和
- 平均值
- 奇偶判断
- 位数处理
- 简单数学公式代入

---

## 四、字符串
### 需要掌握
- 字符串下标访问
- 字符串遍历
- 字符串切片
- `split()`
- `join()`
- `replace()`
- `lower()`、`upper()`
- `ord()`、`chr()`

### 典型题型
- 句子拆分
- 首字母缩写
- 大小写转换
- 字符统计
- 字符位置交换

### 易错点
- 字符串不能原地修改
- 切片右边界取不到
- 字符和字符串混用

---

## 五、列表
### 需要掌握
- 创建列表
- 下标访问
- 修改元素
- `append()`
- `insert()`
- `remove()`
- `pop()`
- `index()`
- `sort()`
- `reverse()`
- `len()`

### 典型题型
- 保存一批数据
- 倒序输出
- 交换位置
- 遍历求和
- 找最大最小值

### 易错点
- 下标越界
- `remove()` 删除的是值，不是下标
- `pop()` 删除的是位置

---

## 六、切片与序列思想
### 需要掌握
- 正向切片：`a[start:end]`
- 步长切片：`a[start:end:step]`
- 逆序：`a[::-1]`
- 字符串、列表都能切片

### 典型用途
- 倒序处理
- 取子串
- 隔位输出
- 快速复制数据

---

## 七、遍历、统计、累计思想
### 需要掌握
- 遍历一个序列
- 用变量维护当前答案
- 边遍历边更新结果

### 常见模式
```python
ans = 0
for x in data:
    ans += x
```

```python
mx = data[0]
for x in data:
    if x > mx:
        mx = x
```

### 典型题型
- 求和
- 求平均
- 找最大值最小值
- 计数
- 统计频率

### 这是很重要的算法基础
后面很多算法，本质上都是这个思想的升级版。

---

## 八、字典
### 需要掌握
- 创建字典
- 添加键值对
- 通过键查值
- `get()`
- `items()`
- `keys()`
- `values()`

### 典型作用
- 建立映射关系
- 统计元素出现次数
- 快速查找某个键对应的值

### 典型题型
- 钥匙与门的对应关系
- 某字符出现多少次
- 根据值反向查找键

### 易错点
- 字典是按键查值，不是按值查键
- 键不存在时直接访问可能报错
- 反查时需要遍历 `items()`

---

## 九、集合
### 需要掌握
- 创建集合
- `in` 判断元素是否存在
- 去重
- 交集、并集、差集的基本概念

### 典型作用
- 判断某个元素是否出现过
- 对数据去重
- 快速成员查询

---

## 十、哈希表思想
这里不是单独某个语法点，而是一种做题思路。

### 需要理解
遇到下面几类题，要想到哈希结构：
- 查某个元素在不在
- 统计频率
- 建立映射关系
- 快速查找

### 常用实现
- 用列表做定长计数
- 用字典做映射和计数
- 用集合做存在性判断

---

## 十一、函数
### 需要掌握
- `def` 定义函数
- 参数
- 返回值 `return`
- 函数调用

### 作用
- 把重复逻辑封装起来
- 让代码更清晰
- 提高复用性

### 典型形式
```python
def add(a, b):
    return a + b
```

---

## 十二、模块与导入
### 需要掌握
- `import math`
- `import sys`
- `__name__ == "__main__"`

### 作用
- 调用现成模块
- 让代码分块组织
- 理解程序入口

---

## 十三、类与面向对象入门
### 需要掌握
- `class`
- `__init__`
- `self`
- 实例属性
- 实例方法

### 作用
- 用类描述“节点”“链表”这类对象
- 为数据结构建模

---

## 十四、链表
### 需要掌握
- 节点 `Node`
- 节点的 `data` 和 `next`
- 链表的头节点
- 遍历链表
- 尾插法建链表
- 查找第 n 个节点
- 插入节点
- 删除节点

### 重点理解
- 链表不是连续存储
- 插入删除靠修改指针
- 头节点常常需要单独处理
- 删除/插入前往往要先找到前一个节点

### 这是正式的数据结构内容
说明已经从“语法题”进入“数据结构入门”。

---

## 十五、栈
### 需要掌握
- 后进先出
- 用列表模拟栈
- `append()` 入栈
- `pop()` 出栈
- `stack[-1]` 取栈顶

### 典型场景
- 需要“最后进去的先处理”

---

## 十六、队列
### 需要掌握
- 先进先出
- 用列表模拟队列
- `append()` 入队
- `pop(0)` 出队
- `queue[0]` 取队头

### 典型场景
- 按进入顺序处理数据

---

## 十七、模拟题能力
### 需要掌握
- 按题意一步一步写
- 不跳步骤
- 根据输入状态不断更新变量

### 常见表现
- 题目不难，但细节特别多
- 重点在于“写对”，不是“想出高深算法”

---

## 十八、边界条件处理
### 需要重点检查
- 空数据怎么办
- 只有一个元素怎么办
- 没找到答案怎么办
- 输入是否结束
- 下标是否合法
- 删除和插入位置是否越界
- 栈或队列是否为空

### 这部分很重要
很多题不是不会做，而是边界没处理好。

---

# 一句话总结主线
这份题单的知识路线是：

**输入输出 → 循环判断 → 字符串/列表 → 统计与查找 → 字典/集合 → 函数 → 类 → 链表 → 栈与队列**

也就是：

**Python写题基础 + 算法入门前的数据结构基础**
